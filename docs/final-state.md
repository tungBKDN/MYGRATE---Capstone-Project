# Final Project State

This repository now keeps the core source code in the existing source folders and removes temporary/generated clutter from the root.

## What was cleaned

- Removed generated bytecode artifacts such as `MigrationStub_*.class`.
- Removed transient cache folders such as `__pycache__`.
- The root `.pytest_cache` folder may still need manual cleanup if it is locked by local tooling or permissions.
- Consolidated repeated ANSI-cleaning logic into `script_utils.py`.
- Refactored the main reporting scripts to use smaller helpers and shared constants.
- Treated generated inspection markdown files as disposable outputs.

## Current layout

- `src/`: application and workflow source code.
- `scripts/reporting/`: reporting and classification entry points.
- `scripts/`: remaining utility entry points and local tooling.
- `artifacts/`: discovery and evaluation artifacts.
- `freshbrew_data/`: dataset workspace snapshots.
- `org_freshbrew_source/`: source mirror used by the migration pipeline.
- `working/`: runtime working directories produced by the migration flow.
- `docs/reports/`: generated inspection reports.
- `docs/`: final documentation and project notes.

## Useful entry points

- `scripts/reporting/eval_summary.py`: prints the migration summary table.
- `scripts/reporting/generate_inspection_reports.py`: generates detailed inspection reports.
- `scripts/reporting/generate_llm_like_reports.py`: generates narrative-style reports.
- `scripts/reporting/classify_by_old_causes.py`: classifies legacy failure causes.
- `run_batch.py`: batch runner for evaluation workflows.
- `simulate_cli.py`: local CLI simulation entry point.

## How to reproduce

1. Run the summary script: `python scripts/reporting/eval_summary.py`
2. Generate inspection reports: `python scripts/reporting/generate_inspection_reports.py`
3. Generate narrative reports: `python scripts/reporting/generate_llm_like_reports.py`
4. Classify old causes: `python scripts/reporting/classify_by_old_causes.py`
